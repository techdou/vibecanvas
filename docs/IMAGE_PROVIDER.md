# Image Provider

## Profile

A provider profile contains endpoint paths, model alias, API and download headers, relay-specific JSON, safety controls, declared capabilities, and cost estimates. Profiles are stored in the shared VibeCanvas config, never in graph JSON.

## Generation

Default JSON endpoint:

```text
POST <baseUrl>/images/generations
```

Fields include model, prompt, quality, output format, and optional custom size. Candidate requests are sequential by default for relay compatibility and independent cancellation.

## Editing

Default multipart endpoint:

```text
POST <baseUrl>/images/edits
```

Inputs:

- source image;
- repeated configurable image field for references;
- annotation image;
- optional `mask`;
- prompt, size, quality, model, output format;
- relay-specific extra fields.

## Response compatibility

Accepted image items:

- `data[].b64_json`
- `data[].url`
- `data[].image_url`
- equivalent `images` or `output` arrays where items expose those fields.

## Safety and validation

- timeout and retry with AbortSignal-aware delays;
- manual redirect handling and maximum three redirects;
- HTTP/HTTPS only;
- no embedded URL credentials;
- private, loopback, link-local, multicast, IPv4-mapped IPv6, and metadata-like addresses blocked by default;
- optional host allowlist;
- 80MB remote output limit;
- 50MB per edit input limit;
- mask must match source dimensions and contain alpha;
- generated bytes must decode as an image, match target ratio within tolerance, and provide sufficient resolution;
- output is normalized to the requested real format before its final extension is chosen.

Only enable private output URLs for a trusted local relay.

## Capabilities

The UI and Agent read declared capability flags:

```text
textToImage
imageToImage
multiReference
maskEdit
customSize
transparentBackground
batchN
maxReferences
maxCandidates
responseFormats
```

These are provider declarations, not automatic proof. Use `npm run doctor -- --probe-provider` to verify connectivity, and test editing separately for non-standard relays.

## Cost table

Per-image estimates for low, medium, high, and auto quality plus an edit multiplier are recorded in Run cost panels. They are estimates supplied by the user, not billing data from the relay unless the Agent response reports an actual cost.
