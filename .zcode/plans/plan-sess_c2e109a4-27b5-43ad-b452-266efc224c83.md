# Canvas Creative — 完整开发方案

## 项目定位

基于 CowArt 架构精华（tldraw + MCP + 文件存储 + widget bridge），从零写一个精简的 AI 图片创作画布。

**核心差异**：CowArt 依赖 Codex 内置 imagegen → 本项目自己直接调 OpenAI 格式 API 生成图片。

## 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 画布 | tldraw v5 | 自由画板 + 标注 + 连线（原生 binding） |
| 前端 | React 19 + Vite 7 | 精简，预计 ~1500 行 |
| MCP | @modelcontextprotocol/sdk + ext-apps | 标准 MCP 工具 + Codex widget bridge |
| 图片 API | OpenAI 格式 `/v1/images/generations` + `/v1/images/edits` | Node 侧调用，Key 不出后端 |
| 存储 | 纯文件系统（per-page JSON + 图片文件） | 同 CowArt，简洁可靠 |

## 四大核心功能

### 1. 文生图（文本节点 → 生成图片）

```
用户添加文本节点 → 输入 prompt → 点"生成"
  → 前端调 MCP 工具 generate_image({ prompt, size })
  → Node 侧：fetch OpenAI API → 拿到 base64/URL → 下载验证 → 写入 page assets
  → 复用 insert_image 逻辑：创建 tldraw image shape，放在文本节点旁边
  → 画布自动刷新（widget: 轮询; 网页: SSE）
```

**新增 MCP 工具**：`generate_image`
- 参数：`prompt`（必填）、`size`（可选，如 "1024x1024"）、`projectDir`、`pageId`、`anchorShapeId`
- 内部：调 OpenAI → 写文件 → 调 insertImage 内部函数 → 返回 shapeId + assetId

### 2. 标注二创（图片标注 → 图生图）

```
用户选中图片 → 标注工具画箭头/写批注 → 点"按标注修改"
  → tldraw editor.toImageDataUrl() 导出标注截图
  → 前端调 MCP 工具 edit_image({ sourceImagePath, annotationImagePath, prompt })
  → Node 侧：调 OpenAI /v1/images/edits（multipart：source + annotation 作为参考）
  → 结果图片放到原图旁边（不删原图，不删标注）
```

**新增 MCP 工具**：`edit_image`
- 参数：`sourceImagePath`、`annotationImagePath`、`prompt`、`projectDir`、`pageId`、`anchorShapeId`

### 3. 节点连线

CowArt 关闭了 tldraw 原生 arrow 工具。本项目**重新启用**，并加语义：
- 用 `meta.ccEdge: true` 标识"生成关系"边
- arrow 自动 binding 到 shape（tldraw 原生吸附）
- 连线表示"这张图是从那张图生成的"——可视化创作链路
- 存储层零改动（tldraw binding 记录自动序列化）

### 4. 迭代修改

```
选中已生成图片 → 右键/面板"重新生成" → 修改 prompt → 生成新图
  → 新图替换原图位置（或放旁边，可选）
  → 自动画一条连线从旧图→新图（可选，表示迭代关系）
```

## 双模式架构

### Widget 模式（Codex 原生体验）

保留 CowArt 的 widget bridge 设计：
- `render_canvas_widget`（App Tool）→ Codex 渲染 widget iframe
- `window.canvasMcp.callServerTool()` → 浏览器反向调 MCP 工具
- `window.canvasMcp.sendFollowUpMessage()` → 画布向 Agent 发消息（可选，用于 Agent 辅助场景）

**关键改动**：widget bridge 里把 `cowartMcp` 改名 `canvasMcp`，品牌从 CowArt 改 Canvas Creative。

### 网页模式（ZCode/Claude/Trae 等所有 Agent）

完整的 localhost 网页 + HTTP API：
- `vite dev server` 提供 HTTP API（画布读写、图片生成、标注二创）
- SSE 推送画布变更
- Agent 通过标准 MCP stdio 工具操作画布（与 widget 模式共享同一套工具）
- 用户在浏览器 `http://127.0.0.1:5180` 看画布

**补全 CowArt 缺失的 HTTP API**：
- `POST /api/generate` → 等价于 MCP `generate_image` 工具
- `POST /api/edit` → 等价于 MCP `edit_image` 工具
- `POST /api/reference-image` → 保存参考图
- `GET /api/page-asset` → 懒加载资源

## MCP 工具清单（8 个）

| 工具 | 类型 | 作用 |
|---|---|---|
| `render_canvas_widget` | App Tool | 打开画布 widget（Codex 专用） |
| `get_canvas_state` | 只读 | 读画布快照 + 视图 + 存储路径 |
| `save_canvas_state` | 写 | 持久化 tldraw 快照 |
| `get_selection` | 只读 | 读选中 shape |
| `generate_image` | **新增** | **调 OpenAI API 文生图 → 自动回插画布** |
| `edit_image` | **新增** | **调 OpenAI API 图生图（标注二创）→ 放原图旁** |
| `insert_image` | 写 | 把本地图片文件插入画布（通用，不调 API） |
| `save_reference_image` | 写 | 保存参考图到 page assets |

## 配置

### 图片 API 配置

环境变量（`.env`）或 MCP server 启动参数：
```
IMAGE_API_KEY=sk-xxx
IMAGE_API_BASE_URL=https://api.openai.com/v1
IMAGE_API_MODEL=gpt-image-1
```

### 多 Agent 配置

| Agent | 配置文件 | 格式 |
|---|---|---|
| Codex | `.codex-plugin/plugin.json` + `.mcp.json` | Codex 私有 |
| ZCode | `.zcode/config.json` → `mcp.servers` | 标准 MCP |
| Claude Code | `.mcp.json` → `mcpServers` | 标准 MCP |
| 通用 | `.agents/mcp.json` → `mcpServers` | 跨工具兼容 |

一个安装脚本 `scripts/setup.mjs` 自动写各 Agent 的配置文件。
